# 端到端测试用例

部署完成 + 配置 `DEEPSEEK_API_KEY` 后，用这 3 条消息验证 AI 输出质量。

每条都应稳定输出：情绪标签 + 60 字内潜台词 + 60 字内真实需求 + 3 套风格不同的话术 + 2-3 条雷区。

---

## Case 1：经典「我没事」（女→男视角）

| 字段 | 输入 |
| --- | --- |
| 视角 | 男生 → 女生 |
| 阶段 | 稳定期 |
| 背景 | 昨天答应陪她逛街，今天临时被同事拉去加班 |
| 对方原话 | 没事，你忙你的吧。 |

**期望输出特征**
- 情绪 ≈ 失望 / 委屈
- 潜台词应捕获「我其实期待你优先选我」
- 三套话术应避免直接道歉式空话，要有具体行动承诺
- 雷区应包含 "好的那我先忙了" / "下次补给你"

---

## Case 2：测试型问题（男→女视角）

| 字段 | 输入 |
| --- | --- |
| 视角 | 女生 → 男生 |
| 阶段 | 暧昧期 |
| 背景 | 第三次约饭结束，回家路上 |
| 对方原话 | 你今晚开心吗？我感觉你有点心不在焉。 |

**期望输出**
- 情绪 ≈ 不安 / 试探
- 潜台词应识别「在确认我的好感是否真实」
- 话术应给具体记忆细节回应、避免「我挺开心啊」式敷衍

---

## Case 3：吵架冷战（任意视角）

| 字段 | 输入 |
| --- | --- |
| 视角 | 男生 → 女生 |
| 阶段 | 磨合/吵架 |
| 背景 | 因为我妈来住几天的事冷战 2 天，今天 TA 发了这条 |
| 对方原话 | 算了我懒得说了，你和你妈一边去吧。 |

**期望输出**
- 情绪 ≈ 委屈 / 生气 / 疲惫
- 真实需求：被理解、被站队
- 三套话术差异明显：温柔承认错误 / 提具体解决方案 / 不严肃但坦诚
- 雷区应警示「就事论事」「冷静一下」之类的 trigger 词

## 限频测试

在同一 IP 连续提交 6 次，第 6 次应返回 429：
> 今日免费额度已用完（5 次/天）。明天再来，或留意付费版上线。
